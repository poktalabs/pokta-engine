# M1 Engine Plan — Mi Pase daily pricing in godin-engine

Generated by /plan-eng-review on 2026-06-08 · Branch: main · Repo: github.com/poktalabs/godin-engine
Design: ./DESIGN.md (APPROVED) · Scope: M1 of 2 (engine core; M2 = Vite SPA + Privy + scheduling)
Status: PLANNED

M1 rewrites the validated Mi Pase daily-pricing pipeline into godin-engine and proves it
end-to-end **via `/v1`** (operator triggers + approves; no UI), writing to the Shopify **dev
store**. Single-tenant (Mi Pase), forward-compatible (`consumer_id` = future `tenant_id`).

## Decisions (from the review)

- **D1 — `onComplete` primitive.** Add manifest `onComplete?: workflowId`: on run success the
  worker auto-dispatches that workflow with the output as input (NO gate), symmetric to `onApprove`.
- **D2 — `ctx.integration(name)` lazy scoped accessor.** Not a fat `ctx.integrations` object —
  returns only the requested provider's configured client, resolved per-tenant by `run.consumer_id`
  (env-backed for M1). Narrow secret blast radius (Codex #5).
- **D3 — `packages/shopify` + `packages/mercadolibre`**, fail-soft like `packages/notion`/`resend`
  (read config, throw-when-unconfigured → caller records IntegrationResult).
- **D4 — Rewrite the shell, keep the brain.** Port the validated pure logic VERBATIM into
  `workflows/pricing/lib/` and carry the **54 tests** as a parity harness. The "rewrite" is scoped
  to orchestration + IO seams; the validated decision logic is unchanged.
- **D5 — `engine_workflow_state` table** for the 1% anti-thrash + resumability (rich schema below).
- **D6 — One batch run** over 316 SKUs; `run.output` = **compact summary + artifact ref**; the
  durable per-SKU record lives in `engine_workflow_state`; the gate artifact = **flagged subset only**.
- **D7 — Per-SKU resumable apply.** Each SKU is checkpointed **before and after** its write; a
  re-run retries **only** failures (idempotent). Partial-failure is a first-class outcome.
- **D8 — Generous `timeoutMs` (~15-20min) + reuse existing pacing + build the reaper** (a standing
  engine gap) so a crashed mid-batch run never strands.
- **D9 — Dev store only.** Margin validation gates the PROD cutover (M-later TODO).

### onComplete + gate semantics (Codex #2 — defined)
- `pricing-draft` run is `succeeded` once it emits output.
- `pricing-apply-confident` (via `onComplete`) and `pricing-apply-flagged` (via `onApprove`) are
  **independent child runs**, each with its own status + per-SKU resumability.
- Approval governs **only** the flagged child. A confident-apply partial failure is recorded on its
  own child (resumable) and does **not** block or alter the flagged gate.

## Data flow

```
POST /v1/workflows/pricing-draft/runs  (operator; consumer_id=mi-pase; input={scope/limit})
      │  one txn: insert engine_runs(queued) + boss.send
      ▼
pricing-draft.run(input, ctx)            runtime: agent/sandbox   timeoutMs ~20min
  ├─ ctx.integration('shopify').getCatalog()          ┐ paced external IO
  ├─ ctx.integration('mercadolibre').search(per SKU)  ┘ (ml-batch pacing/state reused)
  ├─ match (matching-score) → price (pricing-logic 8-branch)   [pure, 54 tests, verbatim]
  ├─ classify via decisionForMatchConfidence → confident[] | flagged[]
  ├─ upsert desired rows → engine_workflow_state(status=pending, desired_price, hash, prior)
  └─ output = { summary{counts,...}, artifactRef }            (full detail in state, not output)
      │
  worker on success:
   ├─ onComplete → pricing-apply-confident   ──── auto child run, NO gate ───────────┐
   └─ approval policy → engine_approvals(pending, artifact=flagged subset)           │
                          │                                                          │
                   [Dalia: POST /v1/approvals/:id/approve]                           │
                          ▼                                                          ▼
                 pricing-apply-flagged child run            pricing-apply-confident child run
   both run pricing-apply (shared):  for each SKU →
     skip if |new−lastApplied| < 1%  (anti-thrash)
     state=attempting → ctx.integration('shopify').updateVariantPrice() → state=applied|failed
     (checkpoint before + after each write; dev store)
     output = { applied:n, skipped:k, failed:m, perSku[] }   ; re-run retries only failed
```

## `engine_workflow_state` schema (D5/D7, Codex #4)

```
engine_workflow_state
  consumer_id      text     ┐ PK
  workflow_id      text     │ (logical: 'pricing')
  sku              text     ┘
  desired_price    numeric        -- last computed target
  desired_hash     text           -- hash(inputs) — detect stale desired
  prior_shopify    numeric        -- value before our write (for rollback/audit)
  attempted_price  numeric
  status           enum(pending|attempting|applied|failed|skipped)
  failure_reason   text
  source_run_id    text           -- which run set this
  updated_at       timestamptz
```

## Failure modes (each new path)
- ML 403 / empty / non-MXN → competitor price empty → SKU flagged (handled; test). Error surfaced in row.
- Shopify 422 / rate-limit mid-batch → per-SKU `failed` + resumable (T8). Visible in run output + state.
- Worker crash mid-batch → checkpoint (before/after) + reaper (T5) → no stranded run; re-run resumes.
- ML OAuth token expiry → refresh in `packages/mercadolibre`; on failure, fail-soft (competitor empty).
- Oversized output → artifact ref + per-SKU state, not a giant `run.output` (D6). 
- **Critical-gap check:** none silent — every failure path has per-SKU state + a surfaced outcome.
