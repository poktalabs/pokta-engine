## Locked decisions

1. **Tenant id canonical = `mi-pase`** (the engine's system-of-record id; in `engine_runs.consumer_id` + SERVICE_KEYS). `engine_tenants.tenant_id == consumer_id`. (SPA route rename `mipase→mi-pase` is a PR2b task.)
2. **Shared tenant-registry module, ~60s in-process TTL cache**, used by BOTH engine-api and worker (each process caches independently). Cache miss reads one indexed row.
3. **Workflow allow-list enforced in engine-api** (keep `@godin-engine/workflows` pure): one helper filters `listManifests()` and gates `POST /v1/workflows/:id/runs` → `SKILL_NOT_FOUND` when not in `tenant.allowed_workflows` (text[]).
4. *(PR2b)* Full Privy P6 in the SPA.
5. **Shared `TenantView` type in `@godin-engine/contract`** = the `GET /v1/tenants/me` response. Engine maps row → TenantView; SPA imports it (PR2b). Includes **typed branding** (not raw jsonb) and the **allow-list-filtered `allowedWorkflows`** so the SPA never calls a forbidden workflow.
6. *(PR2b)* SPA tests via vitest+jsdom.
- **Membership (cross-model #1): `members text[]` of allowed Privy DIDs on the tenant row.** `resolveTenant` for a Privy principal = find the tenant whose `members[]` contains the DID (none → `TENANT_UNKNOWN`; multiple → reject explicitly as ambiguous for now). Logic ships + is tested in PR2 (Privy-mode principal); real Privy tokens come from PR2b.

## `engine_tenants` schema (T5)

```
engine_tenants
  tenant_id        text PK          -- == consumer_id, e.g. 'mi-pase'
  name             text not null
  status           tenant_status not null default 'active'   -- enum: active | pending | disabled
  currency         text not null    -- ISO 4217 (MXN | USD) — DISPLAY-only (does not affect execution)
  locale           text not null    -- es-MX | en — DISPLAY-only
  branding         jsonb not null   -- typed against TenantView.branding (name, badge?)
  allowed_workflows text[] not null default '{}'   -- validated against manifest ids on seed/save
  members          text[] not null default '{}'    -- allowed Privy DIDs
  secret_prefix    text             -- ops-owned; validated ^[A-Z][A-Z0-9_]*$ + UNIQUE
  created_at       timestamptz not null default now()
  updated_at       timestamptz not null default now()   -- (Codex) audit + future invalidation
  index on (members) [gin]  -- DID membership lookup
```

Seed: **mi-pase** `status='active'` (real: allowed_workflows=pricing-*, secret_prefix=MIPASE, members=<mel DID>). **vino** `status='pending'` (NOT fake-live: real GHL/JobTread/Gmail creds + allow-list land in PR3 — seeding it active would create misleading green tests).

## Hardening folded in (from Codex)

- **status enforcement:** `resolveTenant` rejects `disabled`/`pending` → `TENANT_UNKNOWN`; `/v1/tenants/me` 403s a disabled tenant. (Single chokepoint = `resolveTenant`.)
- **secret_prefix:** validated charset + uniqueness on seed/save; treated as ops config, never arbitrary tenant input.
- **allowed_workflows:** validated against real manifest ids (from `listManifests()`) on seed; unknown id = seed error.
- **Stale-authz tradeoff (60s TTL):** documented — disabling a tenant / pulling a workflow / rotating secret_prefix takes effect within the TTL. Acceptable for operator-run MVP. Invalidation path (manual bust) = a PR-later TODO.
- **Worker split-brain / child runs:** invariant documented — child runs (onApprove/onComplete) are chain-internal, NOT independently POST-able (`gatedTargets()` blocks direct POST), and inherit `consumerId` from the parent run (stays in-tenant). The worker re-checks the run's tenant is still resolvable/active at execution (cheap, via the cached registry) before running side effects.
- **Anti-enumeration:** `SKILL_NOT_FOUND` for a disallowed workflow is deliberate (matches PR1's cross-tenant 404 posture) — documented in code.

## Test matrix (PR2 backend — all via service-key path + endpoint)

```
REGISTRY       getTenant cache hit/miss/TTL-expiry; unknown id → undefined
RESOLVE        service-key consumer → tenant; Privy DID in members[] → tenant;
               DID in no members[] → TENANT_UNKNOWN; status disabled/pending → TENANT_UNKNOWN
ALLOW-LIST     mi-pase POST pricing-draft → queued; mi-pase POST a vino workflow → SKILL_NOT_FOUND;
               vino POST pricing-* → SKILL_NOT_FOUND; listManifests filtered per tenant
TENANTS/ME     authed mi-pase → TenantView (typed branding, filtered allowedWorkflows, integrations);
               unauthenticated → 401; disabled tenant → 403; unknown principal → TENANT_UNKNOWN
SECRET_PREFIX  worker provider-config resolves MIPASE from registry (not ENV_PREFIX); validation rejects bad prefix
SEED           allowed_workflows validated vs manifest ids; secret_prefix uniqueness/charset enforced
ISOLATION ★    service-key tenant A cannot read/dispatch as tenant B (PR1 suite stays green after registry swap)
REGRESSION ★   mi-pase M1 pricing chain still green after resolveTenant→registry + allow-list gate (CRITICAL)
```
